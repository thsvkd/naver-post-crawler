# naver-blog-crawler

네이버 블로그의 **전체 글을 과거→최근 순서로 txt 파일로 백업**하는 크롤러입니다.
각 글의 제목·작성 날짜·본문을 글 1개당 파일 1개로 저장합니다.

## 동작 방식

- **목록 수집**: 모바일 블로그 `post-list` JSON API를 페이지네이션하며 전체 글 메타데이터를 모읍니다.
  (`totalCount`가 0으로 고정되어 신뢰할 수 없으므로 `items`가 빈 페이지를 만날 때까지 진행합니다.)
- **입력**: 블로그 아이디(`winter9377`)뿐 아니라 블로그·포스트 URL도 인식해 아이디를 추출합니다.
- **순서**: API는 최신→과거 순이라, 전량 수집 후 뒤집어 과거→최근으로 정렬하고 `0001`부터 번호를 매깁니다.
- **본문 추출**: `PostView` HTML의 `se-main-container`를 파싱합니다. 헤드리스 브라우저는 쓰지 않습니다.
  - 텍스트는 그대로, 텍스트 외 요소는 플레이스홀더로 표기해 순서·맥락을 보존합니다.
    - 이미지 → `[이미지: <URL>]`
    - 링크 카드 → `[링크: <제목> <URL>]`
    - 인용구 → `> ` 들여쓰기
- **빈 글 제외**: "N년 전 오늘 / 그날의 추억" 자동 노출 글은 본문이 없으므로 건너뜁니다.
  API의 `thisDayPostInfo`(선필터)와 본문 모듈 판정(`se-anniversarySection` 단독) 두 신호로 판별합니다.
- **재개(증분 저장)**: 파일명 끝의 `logNo`로 글을 식별해, 이미 받은 글은 다시 받지 않습니다.
  글이 삭제돼 순번이 밀려도 위치가 아닌 글 ID로 판정하며, 어긋난 파일은 현재 순번으로 이름을
  맞춰 정렬을 유지합니다. `--force`로 전체를 다시 받을 수 있습니다.

## 설치

[uv](https://docs.astral.sh/uv/)가 필요합니다.

```bash
uv sync
```

## 사용법

```bash
uv run naver-blog-crawler <BLOG | URL> [옵션]
```

`<BLOG | URL>`은 블로그 아이디, 블로그 URL, 포스트 URL 모두 가능합니다.

```bash
uv run naver-blog-crawler winter9377
uv run naver-blog-crawler https://m.blog.naver.com/winter9377
uv run naver-blog-crawler https://blog.naver.com/winter9377/223189475037
```

### 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `-o, --out` | `output` | txt 파일을 저장할 디렉토리 |
| `--delay` | `0.5` | 요청 사이 대기 시간(초) |
| `--max-retries` | `3` | 요청 실패 시 최대 재시도 횟수 |
| `--limit` | (전체) | 처리할 글 수 제한(과거부터). 시험용 |
| `--force` | off | 이미 저장된 글도 다시 받아 덮어쓰기 |

### 출력 예시

```
output/
  0001_2023-08-21_맥쿼리 인프라 유상증자 입고_223189475037.txt
  0002_2023-08-22_상반기 연금펀드 수익률 31%_223189644114.txt
  ...
```

파일명 끝의 숫자는 글의 `logNo`로, 증분 저장 시 글을 고유하게 식별하는 데 쓰입니다.

파일 내용:

```
제목: 맥쿼리 인프라 유상증자 입고
날짜: 2023-08-21
주소: https://m.blog.naver.com/winter9377/223189475037
============================================================

8월 21일 주식 이야기
...
```
