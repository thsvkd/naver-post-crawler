# naver-post-crawler

네이버 **블로그·카페**의 글을 **과거→최근 순서로 txt 파일로 백업**하는 크롤러입니다.
각 글의 제목·작성 날짜·본문을 글 1개당 파일 1개로 저장합니다.

헤드리스 브라우저 없이 **HTTP 요청 + HTML 파싱**만으로 동작하며, 동일한 코어를 쓰는
**CLI**와 **Flet 기반 데스크톱 GUI**를 함께 제공합니다. 블로그와 카페는 같은
수집·저장 파이프라인을 공유하고, 소스별로 클라이언트와 본문 파서만 갈아 끼웁니다.

## 주요 특징

- **전체 백업**: 블로그의 모든 글(또는 카페의 접근 가능한 게시판 글)을 과거→최근 순으로 정렬해 글 1개당 txt 파일 1개로 저장합니다.
- **블로그·카페 지원**: 입력이 카페 주소(`cafe.naver.com/...`)면 자동으로 카페 모드로 동작합니다. 카페 본문도 대부분 같은 스마트에디터라 본문 파서를 재사용합니다.
- **카페 로그인(쿠키)**: 로그인/등급 제한 게시판은 브라우저에서 얻은 세션 쿠키(`NID_AUT`/`NID_SES`)를 주입해 접근합니다. 쿠키는 문자열로 직접 넣거나, 브라우저 확장으로 내보낸 **쿠키 파일**을 주면 되고, GUI는 "쿠키 업데이트"로 파일을 불러와 내부에 저장합니다. 자동 로그인은 하지 않습니다.
- **유연한 입력**: 블로그 아이디, 블로그 URL, 포스트 URL, 카페 클럽 URL/게시판/글 URL을 모두 인식합니다.
- **증분 재개**: 글의 `logNo`로 식별해 이미 받은 글은 다시 받지 않습니다. `--force`로 전체 재수집도 가능합니다.
- **중단 안전**: 임시 파일에 쓴 뒤 원자적으로 교체하므로 Ctrl-C로 중단돼도 잘린 파일이 남지 않습니다.
- **실패 재시도**: 본문을 받지 못한 글을 자동 재시도하고, 끝내 실패하면 기록해 두었다가 다음 실행 때 대화형으로 다시 시도합니다.
- **콘텐츠 충실도**: 이미지·링크 카드·인용구·동영상·구분선을 플레이스홀더로 표기해 순서와 맥락을 보존합니다.
- **구버전 폴백**: 구버전 스마트에디터(SE 3.0) 글도 폴백 파싱합니다.
- **빈 글 제외**: "그날의 추억" 자동 노출 글은 매직넘버에 의존하지 않고 의미 신호 두 가지로 판별해 건너뜁니다.
- **CLI + GUI**: 같은 코어를 쓰는 명령줄 도구와 데스크톱 앱을 모두 지원합니다.

## 동작 방식

- **목록 수집**: 모바일 블로그 `post-list` JSON API를 페이지네이션하며 전체 글 메타데이터를 모읍니다.
  (`totalCount`가 0으로 고정되어 신뢰할 수 없으므로 `items`가 빈 페이지를 만날 때까지 진행합니다.)
- **입력**: 블로그 아이디(`winter9377`)뿐 아니라 블로그·포스트 URL도 인식해 아이디를 추출합니다.
- **순서**: API는 최신→과거 순이라, 전량 수집 후 뒤집어 과거→최근으로 정렬하고 `0001`부터 번호를 매깁니다.
- **본문 추출**: `PostView` HTML의 `se-main-container`를 파싱합니다. 헤드리스 브라우저는 쓰지 않습니다.
  - 텍스트는 그대로, 텍스트 외 요소는 플레이스홀더로 표기해 순서·맥락을 보존합니다.
    - 이미지 → `[이미지: <URL>]`
    - 링크 카드 → `[링크: <제목> <URL>]`
    - 인용구 → `> ` 들여쓰기
  - `se-main-container`가 없는 **구버전 스마트에디터(SE 3.0) 글**(주로 외부 글을
    가져온 `[공유]` 글)은 `se3_view`의 언더스코어식 컴포넌트(`se_paragraph`·
    `se_oglink` 등)로 폴백 파싱합니다. 글 제목 컴포넌트는 본문에서 제외합니다.
- **빈 글 제외**: "N년 전 오늘 / 그날의 추억" 자동 노출 글은 본문이 없으므로 건너뜁니다.
  API의 `thisDayPostInfo`(선필터)와 본문 모듈 판정(`se-anniversarySection` 단독) 두 신호로 판별합니다.
- **재개(증분 저장)**: 파일명 끝의 `logNo`로 글을 식별해, 이미 받은 글은 다시 받지 않습니다.
  글이 삭제돼 순번이 밀려도 위치가 아닌 글 ID로 판정하며, 어긋난 파일은 현재 순번으로 이름을
  맞춰 정렬을 유지합니다. `--force`로 전체를 다시 받을 수 있습니다.
- **중단 안전(Ctrl-C)**: 글은 임시 파일에 쓴 뒤 원자적으로 교체하므로 중단돼도 잘린 파일이 남지
  않습니다. 중단 시 진행분 실패 기록을 저장하고 깔끔히 종료하며, 다시 실행하면 받은 글은 건너뛰고
  이어서 진행합니다.
- **실패 처리/재시도**: 본문을 받지 못한 글은 그 자리에서 몇 번 다시 받아봅니다(스크랩 글 등에서
  서버가 간헐적으로 본문 없는 응답을 주는 경우 대부분 회복). 그래도 실패하면 `output/.failures.json`에
  기록해 두고, 다음 실행 때 **"이전 실패 N건을 다시 시도할까요?"**를 대화형으로 묻습니다
  (`--retry-failed`/`--no-retry-failed`로 강제 가능, 비대화형 환경은 건너뜀). 재시도에 성공하면
  기록은 자동으로 사라집니다.
- **인터페이스**: 실행 중에는 진행바와 최근 처리 결과 로그가 함께 갱신되는 화면을 표시합니다.
- **로깅**: 요청 재시도·본문 재요청·실패·결과 요약을 터미널과 회전 파일
  (`logs/naver-post-crawler.log`) 양쪽에 기록합니다. 터미널 로그는 진행 화면과 같은
  콘솔을 공유해 진행바 위로 흐르고, 에러는 원인 트레이스백까지 파일에 남깁니다.
  `--log-level`(기본 `INFO`)로 상세도를, `--log-dir`로 위치를 조정합니다.

### 카페 수집

입력이 카페 주소(`cafe.naver.com/...`)면 **카페 모드**로 동작합니다. 블로그와
동일한 수집·정렬·저장·재개 파이프라인을 쓰되, 소스 클라이언트와 본문 파서만
카페용으로 바꿉니다.

- **주소 인식**: 클럽 URL(`cafe.naver.com/mycafe`), 클럽 URL + 글 번호, 신형
  SPA 게시판/글 URL(`.../cafes/{cafeId}/menus/{menuId}` 등), 구형 링크
  (`ArticleList.nhn?search.clubid=...`)를 인식해 clubId·menuId·articleId를 뽑습니다.
  클럽 URL만 주면 카페 홈에서 `g_sClubId`를 파싱해 clubId를 해석합니다.
- **대상 범위**: 미지정 시 **전체글(menuId 0)**을 받습니다. 게시판을 가로질러
  카페의 모든 글(읽기 권한 내)을 최신순으로 주며, 각 글의 실제 게시판(menuId)을
  본문 조회에 씁니다. `--menu`로 특정 게시판만, 단일 글 URL로 그 글 하나만 받을 수도 있습니다.
- **본문**: 내부 글 API(`cafe-articleapi/v3`)의 `contentHtml`을 받습니다. 카페
  본문도 대부분 스마트에디터라 블로그와 같은 `se-main-container` 파서를 재사용하고,
  스마트에디터가 아닌 단순 HTML은 텍스트·이미지만 보존해 폴백합니다.
- **로그인**: 등급 제한·비공개 게시판은 세션 쿠키가 있어야 본문을 받을 수
  있습니다. 자세한 방법은 [카페 백업 (로그인 쿠키)](#카페-백업-로그인-쿠키)를 보세요.

> ⚠️ 카페 내부 API는 **비공식·비문서화**라 경로·응답 구조가 예고 없이 바뀔 수
> 있습니다. 응답 처리는 방어적으로 작성했지만, 네이버 변경에 따라 조정이 필요할
> 수 있습니다.

## 요구사항

- **Python 3.14+**
- **[uv](https://docs.astral.sh/uv/)** (의존성 관리·실행)

네이티브 GUI 빌드에는 플랫폼별 툴체인이 추가로 필요합니다([네이티브 앱 빌드](#네이티브-앱-빌드) 참고).

## 설치

```bash
uv sync
```

셋업 스크립트를 쓰면 의존성 동기화와 pre-commit hook 설치를 한 번에 처리합니다(모든 플랫폼 공통).

```bash
python scripts/setup.py
```

## 사용법

```bash
uv run naver-post-crawler <TARGET> [옵션]
```

실행 스크립트로도 동일하게 실행할 수 있습니다(모든 플랫폼 공통). 대상을 인자로 주면 그대로 CLI로 전달되고, 인자 없이 실행하거나 `--gui`를 붙이면 [GUI 창 모드](#gui-데스크톱)로 실행됩니다(대상은 창에서 입력).

```bash
python scripts/run.py                  # GUI 창
python scripts/run.py <TARGET> [옵션]  # CLI 백업
```

`<TARGET>`은 블로그 아이디·블로그/포스트 URL이거나 카페 주소입니다. 카페 주소면 자동으로 카페 모드로 동작합니다.

```bash
# 블로그
uv run naver-post-crawler winter9377
uv run naver-post-crawler https://m.blog.naver.com/winter9377
uv run naver-post-crawler https://blog.naver.com/winter9377/223189475037

# 카페 (공개 게시판)
uv run naver-post-crawler https://cafe.naver.com/mycafe
uv run naver-post-crawler https://cafe.naver.com/mycafe --menu 5

# 카페 (로그인 필요 게시판 — 세션 쿠키 주입)
uv run naver-post-crawler https://cafe.naver.com/mycafe --cookie "NID_AUT=...; NID_SES=..."
```

### 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `-o, --out` | `output` | txt 파일을 저장할 디렉토리 |
| `--delay` | `0.5` | 요청 사이 대기 시간(초) |
| `--max-retries` | `3` | 요청 실패 시 최대 재시도 횟수 |
| `--limit` | (전체) | 처리할 글 수 제한(과거부터). 시험용 |
| `--retry-failed` / `--no-retry-failed` | (대화형 질문) | 이전에 실패한 글 재시도 여부 강제 |
| `--force` | off | 이미 저장된 글도 다시 받아 덮어쓰기 |
| `--cookie` | (없음) | [카페] 세션 쿠키 문자열 `NID_AUT=...; NID_SES=...`. 로그인/등급 제한 게시판 접근용 |
| `--cookie-file` | (없음) | [카페] 브라우저 확장으로 내보낸 쿠키 파일(cookies.txt/JSON) 경로. `--cookie` 대신 사용 |
| `--menu` | (전체글) | [카페] 특정 게시판만 받을 때의 menuId. 미지정 시 전체글(menuId 0) |
| `--log-dir` | `logs` | 로그 파일을 저장할 디렉토리 |
| `--log-level` | `INFO` | 파일 로그 레벨 (DEBUG/INFO/WARNING/ERROR) |

### 출력 예시

```
output/
  0001_2023-08-21_맥쿼리 인프라 유상증자 입고_223189475037.txt
  0002_2023-08-22_상반기 연금펀드 수익률 31%_223189644114.txt
  ...
```

파일명 끝의 숫자는 글의 `logNo`로, 증분 저장 시 글을 고유하게 식별하는 데 쓰입니다.

파일 내용:

```
제목: 맥쿼리 인프라 유상증자 입고
날짜: 2023-08-21
주소: https://m.blog.naver.com/winter9377/223189475037
============================================================

8월 21일 주식 이야기
...
```

## 카페 백업 (로그인 쿠키)

일부 카페는 **로그인**하고 **해당 카페에 가입**해야 게시판 글을 볼 수 있습니다.
네이버는 자동 로그인을 약관으로 금지하고 캡차·기기 인증으로 막기 때문에, 이
도구는 **자동 로그인을 하지 않습니다.** 대신 브라우저에서 이미 로그인한 세션의
쿠키를 주입해 그 권한으로 공개 API를 호출합니다.

### 방법 A — 브라우저 확장으로 쿠키 파일 내보내기 (권장)

크롬 확장 **[Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc)**
를 씁니다. 오픈소스(MIT)이고 **모든 처리가 브라우저 안에서만** 일어나 외부로 쿠키를
보내지 않습니다. (이름이 비슷한 옛 "cookies.txt"/"Get cookies.txt" 확장은 과거
쿠키 탈취 악성코드로 문제됐던 별개 프로그램이니 혼동하지 마세요.)

1. 위 링크에서 확장을 설치합니다.
2. 크롬에서 `https://cafe.naver.com` 에 접속해 **네이버에 로그인**합니다(닉네임이
   보이는 상태까지).
3. 그 탭에서 확장 아이콘을 클릭하면 쿠키 목록이 뜹니다. `NID_AUT`·`NID_SES`가
   보이는지 확인합니다.
4. 내보내기 형식은 기본값 **Netscape**(또는 JSON) 그대로 두고 **`Export`**(또는
   저장 위치를 고르려면 `Export As`)를 눌러 `*_cookies.txt` 파일로 저장합니다.
   - ⚠️ **"Export All Cookies"는 쓰지 마세요** — 모든 사이트 쿠키가 담깁니다.
     반드시 naver.com 탭에서 `Export`로 해당 사이트 쿠키만 저장합니다.
5. 저장한 파일을 크롤러에 넘깁니다.

**CLI** — `--cookie-file`로 파일 경로를 줍니다(Netscape·JSON 모두 인식):

```bash
uv run naver-post-crawler https://cafe.naver.com/mycafe \
  --cookie-file "C:\\Users\\나\\Downloads\\cafe.naver.com_cookies.txt"
```

**GUI** — **고급 옵션 → 쿠키 업데이트** 버튼을 눌러 쿠키 파일을 고르면, 내부에
저장되어 이후 카페 백업에 자동 사용됩니다("저장된 쿠키: 있음 ✓"으로 표시). 한 번
저장하면 다음부터는 파일을 다시 고를 필요가 없고, 세션이 만료돼 오류가 나면 새
파일로 **쿠키 업데이트**만 다시 하면 됩니다. 저장한 쿠키는 CLI에서도 자동으로
쓰입니다(문자열·파일을 주지 않은 경우).

### 방법 B — 쿠키 문자열 직접 붙여넣기

개발자도구에서 값을 직접 복사해 문자열로 줄 수도 있습니다.

1. 로그인한 상태에서 개발자도구(F12) → **Application** → Storage → Cookies →
   `https://cafe.naver.com` 를 엽니다.
2. `NID_AUT`·`NID_SES` 값을 복사해 `--cookie` 또는 GUI **카페 세션 쿠키** 칸에 붙여넣습니다.

```bash
uv run naver-post-crawler https://cafe.naver.com/mycafe \
  --cookie "NID_AUT=여기에값; NID_SES=여기에값"
```

쿠키 출처 우선순위는 **`--cookie`(문자열) → `--cookie-file`(파일) → 저장된 쿠키**입니다.

### 참고

- 쿠키는 **만료**됩니다. 수집 중 `로그인/권한이 필요` 오류가 나면 다시 로그인해
  쿠키를 갱신(방법 A는 파일 재내보내기 후 **쿠키 업데이트**)하세요. 완전 무인 장기
  운영은 캡차·재인증 때문에 안정적이지 않습니다.
- 쿠키가 없으면 **공개 게시판**만 받을 수 있고, 등급 제한 게시판은 건너뜁니다.
- 쿠키는 로그인 세션 그 자체입니다. **외부에 노출하지 마세요.** 저장된 쿠키는 앱
  내부 저장소(사용자 전용 경로)에 두고 로그 파일에는 기록하지 않습니다.

## GUI (데스크톱)

CLI와 동일한 코어를 쓰는 Flet 기반 데스크톱 앱입니다. 블로그/카페 입력·시작/중단·실시간
진행률·최근 결과 로그·완료 요약을 제공하고, 출력 폴더 선택/열기, 이전 실패 재시도
체크박스, 고급 옵션(딜레이·재시도·force·**카페 세션 쿠키/쿠키 업데이트**·로그 레벨)을 포함합니다.
카페 로그인 쿠키는 [카페 백업 (로그인 쿠키)](#카페-백업-로그인-쿠키)를 참고하세요.

화면 갱신은 이벤트 기반입니다. 결과 로그는 글 한 건의 크롤링이 끝날 때마다 즉시 한 줄씩
쌓이고, 빠르게 바뀌는 상태 텍스트(목록 수집 카운트 등)는 0.2초 렌더 틱으로 합쳐 반영해
화면이 밀리지 않습니다. 크롤링이 시작되면 입력·출력 폴더 선택·찾아보기 버튼이 잠깁니다.

개발 중 실행(소스):

```bash
uv run naver-post-crawler-gui
```

실행 스크립트에 `--gui`를 붙여도 동일하게 GUI 창이 열립니다.

```bash
python scripts/run.py --gui
```

### 네이티브 앱 빌드

빌드 스크립트가 실행한 OS를 감지해 빌드합니다(모든 플랫폼 공통). Flutter SDK는 빌드 시
자동으로 내려받으며, 플랫폼별 네이티브 툴체인도 스크립트가 가능한 한 자동 설치합니다.

```bash
python scripts/build.py
```

| 플랫폼 | 자동 설치 항목 | 결과물 |
|---|---|---|
| Windows | Visual Studio C++ 빌드 도구(vs_installer/winget) | `build\windows\` |
| macOS | CocoaPods(Homebrew) · Xcode는 안내 | `build/macos/` |
| Linux | clang·cmake·ninja·GTK3 등(apt/dnf/pacman) | `build/linux/` |

기본 빌드 결과물은 **실행파일 + DLL + `data/` 폴더**가 한 세트라서 폴더째 배포·실행해야
합니다. 실행파일만 떼면 DLL을 찾지 못해 동작하지 않습니다.

#### 단일 실행파일(`--onefile`)

DLL·폴더 없이 실행파일 하나만 배포하고 싶으면 PyInstaller 기반 `flet pack` 폴백을
씁니다(Flutter/네이티브 툴체인 불필요).

```bash
python scripts/build.py --onefile   # 결과물: dist/ 아래 단일 실행파일
```

단, 백신 오탐·느린 첫 실행(temp에 압축 해제)·큰 용량 등 단점이 있고, 위의 포터블
`storage/` 저장 동작은 적용되지 않아 flet 기본 데이터 경로(`<Documents>/flet/<app>`)를
씁니다. 배포보다 임시 실행용 폴백으로 권장합니다.

자세한 사전 준비는 `scripts/build.py` 주석을 참고하세요.

## 프로젝트 구조

```
naver-post-crawler/
├── src/
│   ├── main.py                     # flet build 진입점 셔임
│   └── naver_post_crawler/
│       ├── cli.py                  # CLI 진입점 (click, 블로그/카페 라우팅)
│       ├── gui.py                  # Flet 데스크톱 GUI
│       ├── crawler.py              # 수집 오케스트레이션 (제너레이터, 소스 비의존)
│       ├── source.py               # PostSource 프로토콜 (블로그·카페 공통 계약)
│       ├── http.py                 # 공유 재시도/백오프 GET 헬퍼
│       ├── client.py               # 블로그 HTTP 클라이언트 (httpx)
│       ├── cafe_client.py          # 카페 내부 API 클라이언트 (+ 쿠키 세션)
│       ├── cafe_ref.py             # 카페 URL 해석 (clubId/menuId/articleId)
│       ├── cookie.py               # 쿠키 파일 파싱(cookies.txt/JSON) + 내부 저장
│       ├── parser.py               # HTML → 본문 파싱 (SE 본문 + SE 3.0 + 카페 폴백)
│       ├── writer.py               # txt 파일 저장 (원자적 교체·증분 재개)
│       ├── failures.py             # 실패 기록 저장소 (.failures.json)
│       ├── blog_id.py              # 블로그 아이디/URL 해석
│       ├── models.py               # 데이터 모델
│       ├── enums.py                # 공유 열거형
│       ├── errors.py               # 공유 예외 타입
│       └── log.py                  # 로깅 설정
├── scripts/                        # setup·run·build·test 자동화 스크립트
├── tests/                          # pytest 테스트
└── docs/SPEC.md                    # 기능 명세서
```

## 주의사항

- 이 도구는 **본인이 접근 가능한 글을 개인적으로 백업**하는 용도를 전제로 합니다.
  카페의 경우 **본인이 가입한 카페의, 본인이 볼 수 있는 글**만 대상으로 하세요.
  네이버 이용약관과 저작권, 그리고 글 작성자의 권리를 존중해 사용하세요.
- 서버에 부담을 주지 않도록 요청 사이 대기(`--delay`, 기본 0.5초)를 유지하고, 과도하거나
  대량의 자동 요청은 삼가세요. 네이버는 요청 속도 제한·봇 차단을 운영하며, 과도한
  요청은 일시 차단으로 이어질 수 있습니다.
- 카페 내부 API 사용과 비공개·등급 제한 영역의 자동 수집은 네이버 이용약관 위반 소지가
  있으며, robots.txt 위반이 문제된 판례도 있습니다. 법적 책임은 사용자에게 있습니다.
- 세션 쿠키(`NID_AUT`/`NID_SES`)는 로그인 그 자체이므로 외부에 노출하지 마세요.
- 수집한 콘텐츠의 저작권은 원저작자에게 있습니다. 무단 재배포·상업적 이용에 따른 책임은
  사용자에게 있습니다.
